Profile & Trade Social

Reimagining the Member Experience for TITM's Premium Platform

Status: Feature Proposal
Version: 1.0
Target: Q1 2026

Executive Summary

The current Profile page is a basic settings interface featuring avatar management, tier badges, Discord role synchronization, and account controls. However, there are no social features currently enabled‚Äîdespite the shared_trade_cards table existing in the database with unused capacity.

With the recent completion of four major features‚ÄîTraining (Academy with XP, ranks, achievements, trade card generation), AI Coach (sessions, market analysis, position tracking), Journal (full trade logging with AI grading, analytics, equity curves), and Discord Integration (role sync, RBAC)‚Äîwe now have an incredible data foundation to build a truly premium Profile Hub and Trade Social experience.

The Opportunity

Transform the Profile page from a basic settings interface into a comprehensive trading identity & community hub that showcases member achievements, tracks trading performance, and enables premium social features‚Äîall while respecting privacy and maintaining the quiet luxury aesthetic.

The Foundation

Academy ranks, XP systems, AI grading, trade journals, discord roles, achievement badges, and trade card generation are all complete. We simply need to wire them together into a cohesive, premium UI experience.

Current State

The existing Profile page is a straightforward settings interface. While functional, it doesn't leverage the wealth of data now available across Academy, Journal, AI Coach, and Discord.

Data Architecture

New database tables and extensions required to support Profile Hub and Trade Social features.

Table Name Purpose Key Fields
member_profiles Extended profile data for Trader Identity Card section user_id, bio, tagline, custom_avatar, privacy_settings, whop_affiliate_url, verified_at
social_feed_items Feed entries linking trades, achievements, milestones id, user_id, item_type (trade/achievement/milestone), reference_id, created_at, visibility
social_likes Likes on feed items id, user_id, feed_item_id, created_at
social_comments Comments on feed items (future) id, user_id, feed_item_id, content, created_at, edited_at
affiliate_referrals Referral tracking for WHOP integration id, referrer_id, referred_id, status, whop_data, created_at, converted_at
leaderboard_snapshots Periodic computed leaderboard data id, period (weekly/monthly/alltime), category, user_id, rank, value, snapshot_date
profile_views Analytics: who viewed whose profile id, viewer_id, profile_user_id, created_at

Integration Points

  • Academy System: Pull user_xp, ranks, achievements from existing tables
  • Journal System: Link journal_entries to social_feed_items; compute stats via RPC
  • AI Coach: Pull grades, recommendations for trading transcript display
  • Discord Integration: Sync roles, webhook for trade sharing notifications
  • Trade Card Generation: Reuse existing Satori + Resvg pipeline for feed card templates
  • WHOP API: REST endpoints for membership status, referral tracking, earnings

    Technical Implementation Notes

    Stats Computation

    All metrics in Trading Transcript are computed server-side via Supabase RPC functions. This ensures data integrity and prevents user manipulation. Functions aggregate journal_entries, user_xp, and ai_coach_sessions in real-time.

    Trade Card Generation

    Reuse existing Satori + Resvg pipeline. Pass trade data + selected template to image generation service. Store generated images in Supabase Storage. Return CDN URLs for feed display.

    Feed Pagination

    Use cursor-based pagination via Supabase. Each feed load fetches 20 items sorted by created_at desc. Cursor points to last item's ID. Prevents offset issues at scale.

    Privacy Controls

    Implement RLS policies on social_feed_items, member_profiles. Privacy settings stored in member_profiles table. Enforce at query level. Options: public, private, friends-only.

    Leaderboard Performance

    Pre-compute daily snapshots via scheduled edge function. Store in leaderboard_snapshots. Queries read from snapshots (no live aggregation). Updates overnight during low-traffic windows.

    WHOP Integration

    Use WHOP's REST API + webhooks. Sync membership status on login. Listen for referral conversions via webhook. Store in affiliate_referrals table. Earnings computed from WHOP API responses.
