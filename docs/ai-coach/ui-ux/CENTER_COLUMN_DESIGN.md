## Overview

The **Center Column** is the primary data visualization canvas of the AI Coach interface, occupying 70% of the screen width. It displays context-driven content based on the user's conversation with the AI, including charts, dashboards, tables, and analytical views.

### Design Principles

1. **Context-Driven**: Content changes based on conversation with AI
2. **Data-Dense but Scannable**: Options traders want maximum information without clutter
3. **Interactive**: Click, zoom, filter, time-scrub capabilities
4. **Real-Time**: WebSocket-driven updates for live market data
5. **Persistent**: Doesn't disappear when chat continues (unless user requests new content)
6. **Multi-Panel Capable**: Can split into 2-4 views simultaneously

## Center Column Behavior Rules

### Content Persistence
- **Default**: Content stays visible as chat continues (doesn't disappear)
- **Override**: New AI command replaces content ("Show me NDX chart instead")
- **Multi-panel**: Can add additional panels without replacing existing

### Context Switching
When AI says:
- **"Here's the chart"** â†’ Replace center panel with chart
- **"Let me also show you..."** â†’ Add second panel (split view)
- **"Going back to..."** â†’ Return to previous view

### Real-Time Update Priority
1. **Critical**: Price data, P&L (update every 1 second)
2. **High**: Greeks, levels proximity (update every 30 seconds)
3. **Medium**: Volume, indicators (update every 1 minute)
4. **Low**: Long-term charts, static content (no auto-update)

### Mobile/Responsive Behavior
- **Desktop (>1440px)**: Chat 30% | Center 70%
- **Laptop (1024-1440px)**: Chat 35% | Center 65%
- **Tablet (768-1024px)**: Stack: Chat on top, Center below (toggle)
- **Mobile (<768px)**: Full-screen chat, Center opens as modal
