## 2. Marketing Angle & Messaging Strategy

### 2.1 The Core Hook

The marketing should lead with **proof, not promises**. The February 2026 numbers are the strongest asset. The messaging framework follows a simple hierarchy:

**Proof** (the numbers) → **Access** (what they get) → **Risk Removal** (the price and duration)

### Primary Headline Options

1. *"81% Win Rate. 6:1 Profit Factor. See For Yourself — $49.99 for 7 Days."*
2. *"7 Days. Full Access. $49.99. Our February Track Record Speaks for Itself."*
3. *"Stop Watching From the Sidelines. 7-Day Trial — $49.99."*

### 2.2 Messaging Pillars

| Pillar | Message | Where to Use |
|--------|---------|-------------|
| **Proof** | 81% win rate, 6:1 profit factor in Feb 2026. Real results, not hypotheticals. | Hero banner, stats section, floating CTA |
| **Access** | Full Core Sniper SPX alerts, morning watchlist, educational commentary, community. | Pricing card, feature callouts |
| **Risk Removal** | $49.99 for 7 full days. No commitment. Cancel anytime. 75% off prorated monthly. | CTA buttons, pricing card, floating bar |
| **Urgency** | Limited trial availability for new members. Market conditions evolve—act while edge is proven. | Floating CTA, final CTA section |
| **Social Proof** | Join traders who averaged 100%+ returns per trade with exact entries and exits. | Testimonial area, stats section |

### 2.3 Conversion Psychology

The **$49.99 price point** is strategically chosen. At roughly $7/day, it falls below the psychological threshold where most traders hesitate. Meanwhile, the 6:1 profit factor means a single successful trade during the trial could cover the cost many times over. This creates a compelling risk/reward equation that mirrors what the service itself teaches.

The **7-day window** is also intentional: it covers a full trading week (5 market days), giving new members enough exposure to see the morning watchlist, live SPX alerts, and educational commentary in real market conditions—not a cherry-picked demo.

## 3. Strategic Site Placement Locations

Based on the current homepage architecture, I recommend **5 distinct placement zones** for the trial promotion. Each targets a different stage of the visitor's decision journey.

### Placement 1: Hero Section — Trial Callout Banner

| | |
|---|---|
| **Location** | Directly below the hero headline and above the existing "JOIN NOW" button |
| **Format** | Horizontal glass-card banner (promo-hero-banner style) |
| **Visibility** | New visitors only (hide for authenticated/existing members) |
| **Priority** | 🔴 HIGH — First thing visitors see after the value proposition |

**Rationale:** The hero is the highest-traffic, highest-attention zone. A trial banner here catches visitors before they scroll and presents the lowest-friction entry point immediately. The existing "JOIN NOW" button scrolls to the $199+ pricing section—the trial banner gives fence-sitters a softer alternative.

### Placement 2: Pricing Section — Trial Pricing Card (Position 0)

| | |
|---|---|
| **Location** | Pricing grid, positioned as the **FIRST card** (left-most) before Core Sniper |
| **Format** | Full pricing card matching existing tier card design with emerald "NEW" badge |
| **Visibility** | New visitors only (hide for existing members viewing the pricing section) |
| **Priority** | 🔴 CRITICAL — This is where purchase decisions happen |

**Rationale:** When a visitor reaches the pricing section, they are evaluating commitment levels. The trial card positioned first creates a natural **price-anchoring effect**: $49.99 feels effortless next to $199, $299, and $499. It also captures visitors who want to try before committing. The grid would expand from 3 cards to 4 on desktop (responsive stacking on mobile).

### Placement 3: Floating Bottom Bar (Persistent CTA)

| | |
|---|---|
| **Location** | Fixed position at bottom of viewport, appears after scrolling past hero |
| **Format** | Slim glass bar with emerald accent, trial price, and "Try Now" button |
| **Visibility** | New visitors only, dismissable with close button (session cookie) |
| **Priority** | 🟡 HIGH — Persistent visibility throughout the entire page scroll |

**Rationale:** The floating bar ensures the trial offer remains visible as visitors scroll through features, testimonials, and other content. It acts as a gentle, persistent reminder without being intrusive. The close button respects user choice, and the session cookie prevents it from reappearing if dismissed.

### Placement 4: Stats Section — Performance Proof Card

| | |
|---|---|
| **Location** | Below the existing 4-stat grid (Win Rate, Avg Weekly Gain, etc.) |
| **Format** | Wide glass card showcasing February 2026 results with trial CTA |
| **Visibility** | All visitors (proof builds trust for existing members too) |
| **Priority** | 🟢 MEDIUM — Reinforces credibility with time-specific data |

**Rationale:** The stats section already establishes credibility with general numbers (87% win rate, 100%+ avg gain). Adding a time-stamped February 2026 performance card beneath it adds specificity and recency. General stats can feel vague; dated results feel real and verifiable. This is where skeptical visitors convert from "interesting" to "I should try this."

### Placement 5: Final CTA Section — Trial-Specific Close

| | |
|---|---|
| **Location** | The existing "Stop Missing Winning Trades" section near the footer |
| **Format** | Add a secondary CTA button alongside the existing "Choose Your Plan" button |
| **Visibility** | New visitors (show trial CTA) vs. existing members (show upgrade CTA) |
| **Priority** | 🟢 MEDIUM — Captures visitors who scrolled the entire page without converting |

**Rationale:** Visitors who reach the bottom of the page without converting are either highly skeptical or need a lower-commitment option. The trial CTA here serves as a safety net: "Not ready for $199/month? Try 7 days for $49.99." It reframes the decision from "yes or no" to "full commitment or trial"—both are a win.

## 6. Recommended Implementation Approach

When approved, the implementation should follow the project's standard slice-based delivery process. Here is the recommended slice order:

1. **Whop Product Setup:** Confirm the $49.99 7-day trial plan is active in Whop with the correct checkout link. Verify webhook events for trial start/end.

2. **Trial Pricing Card Component:** Add a "trial" tier to the PricingCard component with emerald theming and NEW badge. Add to the pricing grid as position 0.

3. **Hero Banner Component:** Create TrialHeroBanner component with the stats, price, and CTA. Conditionally render for unauthenticated visitors only.

4. **Floating CTA Bar:** Create FloatingTrialBar with scroll-triggered appearance, close/dismiss with session cookie, and responsive design.

5. **Stats Section Enhancement:** Add February 2026 performance proof card below the existing stats grid with trial CTA.

6. **Final CTA Update:** Add secondary trial CTA button to the "Stop Missing Winning Trades" section for unauthenticated visitors.

7. **Audience Targeting:** Implement new-visitor-only logic using auth state (Supabase session check). Hide trial promotions for authenticated/existing members.

### 6.1 Technical Considerations

1. The pricing grid currently uses a 3-column layout. Adding a 4th card requires updating the grid from `lg:grid-cols-3` to `lg:grid-cols-4`, or using a responsive scroll/carousel on smaller screens.

2. The floating CTA bar should not overlap the existing mobile bottom navigation. A bottom offset should be added for mobile viewports.

3. The Whop checkout link (`plan_NeOYhHAwtWczQ`) should be centralized in the pricing tier config/database rather than hardcoded, allowing easy updates.

4. Performance stats (81%, 6:1) should be stored as updateable config values (env vars or DB) so they can be refreshed monthly without code deploys.
